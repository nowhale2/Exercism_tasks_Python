1. Добавьте товар(ы) в корзину пользователя
MVP должен позволять пользователю добавлять товары в корзину. Это может быть как один товар, так и несколько. Поскольку это минимально жизнеспособный продукт, количество товаров обозначается повторами. Если пользователь хочет добавить 2 апельсина, слово «апельсины» появится в итерации ввода дважды. Если товар уже есть в корзине пользователя, его количество в корзине должно увеличиться на 1. Если товар новый в корзине, его количество должно быть равно 1.

Создайте функцию add_item(<current_cart>, <items_to_add>), которая принимает в качестве аргументов словарь корзины и любой итерируемый объект, похожий на список, с элементами для добавления. Функция должна возвращать новый/обновленный словарь корзины пользователя.


2. Считывайте данные из списка товаров в приложении «Заметки пользователей»
Ой-ой. Похоже, команда разработчиков занимается расширением функционала. Они хотят добавить в MVP дополнительные функции. Теперь приложение должно создавать корзину, считывая данные из приложения «Заметки пользователей». Удобно для пользователей, но требует чуть больше усилий от команды.

Создайте функцию read_notes(<notes>), которая может принимать в качестве аргумента любой итерируемый объект, похожий на список. Функция должна анализировать элементы и создавать пользовательскую корзину/словарь. Каждый элемент должен добавляться в количестве 1. Затем функция должна возвращать новую пользовательскую корзину.


3. Обновите раздел «Идеи» с рецептами
В приложении есть раздел «Идеи», в котором собраны готовые рецепты из разных кухонь мира. Пользователь может выбрать любой из этих рецептов, и все необходимые ингредиенты автоматически добавятся в его корзину. Руководитель проекта попросил вас добавить возможность редактировать эти рецепты из раздела «Идеи», поскольку команда по работе с контентом постоянно меняет ингредиенты и их количество.

Создайте функцию update_recipes(<ideas>, <recipe_updates>), которая принимает в качестве аргументов словарь "ideas" и итерируемый объект с обновлениями рецептов. Функция должна возвращать новый/обновленный словарь "ideas".

4. Сортировка товаров в корзине пользователя
После того как пользователь сформировал корзину, приложение позволяет ему отсортировать товары в алфавитном порядке. Это упрощает поиск и помогает избежать ошибок при вводе данных, например, когда в базе данных встречаются слова «potatoes» и «Potato».

Создайте функцию sort_entries(<cart>), которая принимает в качестве аргумента корзину/словарь и возвращает новый словарь, отсортированный в алфавитном порядке.


5. Отправка корзины пользователя в магазин для выполнения заказа
Приложение должно отправить корзину конкретного пользователя в магазин для выполнения заказа. Однако покупателям в магазине нужно знать, в каком отделе находится нужный товар и нужно ли его хранить в холодильнике. Поэтому (довольно произвольно) «корзину для выполнения заказа» нужно отсортировать в обратном алфавитном порядке, указав количество товара, а также информацию о его местонахождении и необходимости хранения в холодильнике.

Создайте функцию send_to_store(<cart>, <aisle_mapping>), которая принимает на вход корзину пользователя и словарь с номерами проходов в магазине и True/False для холодильного оборудования, необходимого для каждого товара. Функция должна return формировать комбинированную «корзину для выполнения заказа», в которой для каждого товара, который заказывает покупатель, указаны (количество, проход и необходимость холодильного оборудования). Товары должны располагаться в обратном алфавитном порядке.


6. Обновляйте ассортимент магазина в соответствии с тем, что заказал пользователь.
Приложение не может бесконечно принимать заказы от клиентов. В конце концов в магазине закончатся те или иные товары. Поэтому в вашем MVP-приложении нужно обновлять ассортимент магазина каждый раз, когда пользователь отправляет заказ. В противном случае клиенты будут заказывать товары, которых на самом деле нет в наличии.

Создайте функцию update_store_inventory(<fulfillment_cart>, <store_inventory>), которая принимает на вход «корзину для выполнения заказов» и данные о запасах в магазине. Функция должна уменьшить количество товаров в магазине на количество товаров, «заказанных» в «корзине для выполнения заказов», а затем вернуть обновленные данные о запасах в магазине. Если количество товара в магазине становится равным 0, вместо него должно отображаться сообщение «Нет в наличии».
